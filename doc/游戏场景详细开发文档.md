# 游戏场景详细开发文档

更新时间：2026-05-28

本文档用于指导后续“直播弹幕战”游戏场景的详细开发。后续场景相关需求优先按本文档拆任务、验收和提交。

## 1. 场景定位

本项目不是传统写实 RTS，也不是纯展示场景，而是竖屏直播弹幕战的自动战斗舞台。场景要服务三件事：

1. 观众一眼看懂谁在打谁。
2. 大量单位、炮弹和特效同时出现时不卡顿。
3. 中世纪村庄、现代坦克/飞机、士兵和怪物的混搭看起来有统一的舞台逻辑。

推荐基调：中世纪村庄被卷入“玩具沙盘式”的荒诞战争。不要追求过度写实的灰暗战场，要让单位轮廓、道路、建筑阻挡和爆炸焦点清楚。

## 2. 当前技术约束

- 主场景由 `Assets/Scripts/ApocalypseKingUnityGame.cs` 运行时生成。
- 逻辑坐标范围大致为：`x = -360 ~ 360`，`z = -640 ~ 640`。
- `ToWorldPoint(x, z, height)` 会把逻辑坐标乘以 `LogicalToWorld = 0.025f` 转为 Unity 世界坐标。
- 村庄资源来自 Quaternius `MedievalVillageMegaKit`，路径为 `Assets/Resources/Quaternius/MedievalVillageMegaKit`。
- 单位移动不是 Rigidbody/Collider 驱动，不能只靠 Unity Collider 做阻挡。
- 可见建筑和逻辑阻挡区必须同源创建：新增建筑时同步 `AddBuildingObstacle()`。
- 通行道路使用 `AddRoadCorridor()` 记录，单位绕障优先贴近道路。

## 3. 场景总布局

场景按“左侧人族防线 -> 中央村庄战场 -> 右侧怪物入口”组织。竖屏镜头下，主路和战线应形成轻微对角线，避免画面像平铺棋盘。

| 区域 | 推荐逻辑范围 | 功能 | 开发要点 |
| --- | --- | --- | --- |
| 人族集结区 | `x -360~-210`, `z -340~170` | 士兵复活、步兵阵线、蓝色阵营提示 | 放低矮掩体、旗帜、补给箱，不放高建筑遮挡士兵 |
| 坦克重装道 | `x -340~160`, `z -580~-390` | 坦克主推进路线 | 道路要宽，建筑阻挡半径要留足，避免坦克卡住 |
| 村庄主街 | `x -260~245`, `z -280~-105` | 战斗主视觉导线 | 用泥路/车辙贯穿，炮弹和爆炸主要发生在这里 |
| 市场广场 | `x -90~180`, `z -135~85` | 中央交战焦点 | 适合放井、摊位、低矮道具，但不能堵住通行带 |
| 建筑边框区 | 主路两侧 | 塑造村庄体量和纵深 | 高建筑放边缘，留出主路、广场和怪物入口 |
| 怪物入口 | `x 220~360`, `z -335~-150` | 怪物压入村庄的位置 | 可以做焦土、破门、红色环境提示，入口必须足够宽 |
| 飞行走廊 | `z 245~520` | 飞机巡航和投弹 | 不放高塔穿过航线；后续可按建筑高度决定是否绕飞 |

## 4. 视觉风格

### 色彩

- 地面：低饱和草绿、泥土棕、石灰灰。
- 村庄：木色、石色、暗红/深褐屋顶。
- 人族：蓝色 UI 提示和少量冷色信标。
- 怪物：红橙色入口、受击和危险提示。
- 爆炸：黄色、橙色、白热核心为主。

避免让紫色成为默认爆炸色。紫色只保留给明确的魔法、怪物能量或特殊技能。

### 资产统一

现代坦克、飞机和中世纪村庄风格本身有冲突，所以要通过“沙盘战场”统一：

- 材质整体保持风格化，不追求真实金属脏污。
- 地面和道路像一个清晰的战斗棋盘。
- 建筑是舞台边框和障碍物，不是需要完整探索的开放世界。
- 特效可以华丽，但要按强弱分级，不要每次命中都像终极技能。

## 5. 建筑与通行规则

新增任何建筑、塔楼、井、市场棚、围栏时，必须按下面规则判断：

| 类型 | 是否阻挡 | 是否调用 `AddBuildingObstacle()` | 说明 |
| --- | --- | --- | --- |
| 房屋、塔楼、谷仓、教堂 | 是 | 是 | 阻挡范围要比视觉模型略大 |
| 井、固定市场摊、厚围栏 | 是 | 视大小决定 | 如果坦克/怪物可能撞上，就必须登记 |
| 草丛、小石头、木桶、旗帜 | 否 | 否 | 只做装饰，不影响寻路 |
| 泥路、车辙、焦土、弹坑贴片 | 否 | 否 | 不能再制造“地上多余方块” |
| 入口门、桥、路障 | 视设计决定 | 是/否 | 如果是可通行门洞，阻挡要拆成两侧 |

关键原则：

- 视觉上能挡路的东西，逻辑上也要挡路。
- 视觉上只是地面装饰的东西，逻辑上不能挡路。
- 主路、广场、怪物入口、坦克重装道必须保持连续通行。

## 6. 单位路径规则

### 士兵

- 士兵半径小，可以走较窄道路。
- 士兵应跟随目标转向，枪口方向和身体朝向保持一致。
- 士兵不应被小装饰物阻挡。

### 坦克

- 坦克只走宽路和重装道。
- 坦克避障半径要明显大于士兵。
- 坦克炮塔可以瞄准目标，但车体应朝移动方向。
- 建筑密集区必须给坦克留出绕行空间。

### 飞机

- 当前飞机也使用平面避障，避免视觉上穿过建筑。
- 后续如果要让飞机飞越低矮建筑，需要给 `BuildingObstacle.Height` 加高度判定。
- 飞机投弹路径应避开 UI 核心文字区域。

### 怪物

- 怪物从右侧入口压入，优先贴近 `MonsterGate`、`MainStreet`、`MarketPlaza` 这类通行带。
- 怪物半径最大，建筑间距要比视觉上看起来更宽。
- 怪物不能被连续建筑卡死；如有卡住，需要新增道路通行带或扩大入口。

## 7. 镜头与画面构图

默认镜头应保持竖屏可读：

- 采用 3/4 斜俯视，而不是纯俯视。
- 画面中心落在村庄主街和市场广场之间。
- 人族和怪物两侧都要能看到一部分，观众能理解双方推进关系。
- 高建筑只放在边缘或远景，不遮挡主战斗区。
- 爆炸和死亡特效可以短暂抢焦点，但不能长期覆盖怪物或 UI。

相机抖动规则：

- 普通步枪不开启相机抖动。
- 坦克炮、炸弹、怪物重击、死亡大爆炸可以抖动。
- 抖动持续时间短，幅度按武器等级分层。

## 8. 光照、阴影与氛围

短期继续使用内置渲染管线，不急着迁移 URP/HDRP。

优先级：

1. 方向光角度让建筑、坦克和怪物有明确明暗面。
2. 环境光不要过黑，竖屏直播画面要清楚。
3. 单位脚底需要阴影或 Blob Shadow，避免漂浮感。
4. 爆炸和枪口火光可用点光源短暂增强，但要控制数量和生命周期。
5. 雾色和远景暗化用于增加纵深，不要把主战斗区压暗。

## 9. 性能预算

当前场景会同时出现：

- 士兵约 100 个。
- 坦克约 16 个。
- 飞机约 3 个。
- 怪物约 10 个。
- 弹丸上限 `MaxProjectiles = 220`。
- 特效上限 `MaxEffects = 48`。
- 死亡残骸上限 `MaxDeathVisuals = 56`。

后续场景开发必须遵守：

- 不在战斗高峰频繁 `Instantiate` / `Destroy`。
- 新增重复建筑或道具时复用材质，并开启可用的 GPU Instancing。
- 不使用复杂 MeshCollider 作为主要避障。
- 不把大量动态 UI 和静态 UI 放在同一个 Canvas 下。
- 大型粒子效果要分级，普通命中不能占用大爆炸预算。

## 10. 开发阶段

### 阶段 1：场景骨架

目标：把村庄主路、广场、怪物入口、人族防线和坦克重装道整理成清晰布局。

任务：

1. 重排 `CreateVillagePaths()` 的主路、广场和入口。
2. 调整 `CreateMegaKitMedievalVillage()` 的建筑分布。
3. 确保所有主通行带连续。
4. 构建验证并截图检查。

验收：

- 主路、广场、入口在竖屏下清楚可见。
- 士兵、坦克、怪物有足够空间移动。
- 没有地面多余方块影响画面。

### 阶段 2：建筑阻挡与绕路

目标：让单位不会穿建筑，也不会因为建筑太密集而卡死。

任务：

1. 每栋可阻挡建筑同步 `AddBuildingObstacle()`。
2. 每条可通行道路同步 `AddRoadCorridor()`。
3. 怪物、坦克、士兵分别检查避障半径。
4. 用探针或运行日志观察 `buildingOverlaps`。

验收：

- `buildingOverlaps = 0`。
- 怪物遇到房子会绕，不会直穿。
- 坦克不会挤在狭窄建筑夹缝里。

### 阶段 3：视觉层级

目标：让观众在混战中看清主角、危险和结果。

任务：

1. 调整建筑高度和位置，避免遮挡怪物和坦克。
2. 给人族侧、怪物侧、中央战斗区建立颜色提示。
3. 控制装饰物密度，保留主战斗区留白。
4. 爆炸、枪口、死亡特效按强度分层。

验收：

- 画面第一眼能看到主战斗区。
- 怪物、坦克、士兵轮廓不被建筑淹没。
- 特效华丽但不乱。

### 阶段 4：氛围细化

目标：把场景从“摆放正确”推进到“有故事感”。

任务：

1. 怪物入口增加破门、焦土、裂痕、火光。
2. 人族侧增加旗帜、补给、战壕感低矮道具。
3. 村庄中央增加井、市场摊、路牌等低矮生活道具。
4. 炮击区域增加弹坑贴片、烟尘和毁坏痕迹。

验收：

- 村庄像被战争卷入，而不是随机堆建筑。
- 道具不会阻挡主路。
- 低矮装饰不造成寻路问题。

### 阶段 5：性能与验证

目标：保证场景升级后仍能稳定运行。

任务：

1. 检查对象池预热是否覆盖新增特效。
2. 检查运行时材质复用和 instancing。
3. Unity Windows Player 构建验证。
4. 清理截图、日志、临时文件。
5. 中文提交并 push。

验收：

- 构建成功。
- 无临时文件进入提交。
- 场景文件没有只因 Unity 自动重写 fileID 产生的噪声改动。

## 11. 每次开发检查清单

每次新增或调整场景元素，都按这个清单检查：

- 是否影响主路、广场、入口或坦克重装道通行？
- 如果是建筑，是否同步添加或调整 `AddBuildingObstacle()`？
- 如果是道路，是否需要同步添加或调整 `AddRoadCorridor()`？
- 是否遮挡士兵、坦克、怪物、飞机或关键 UI？
- 是否引入新的材质、贴图或资源许可证记录？
- 是否需要对象池预热？
- 是否运行 Unity 构建或合适的运行验证？
- 截图、日志、下载临时文件是否已删除？
- 是否中文提交并 push？

## 12. 后续优先任务

推荐按以下顺序继续开发：

1. 重排村庄主街、市场广场、怪物入口，让路线更自然。
2. 给怪物入口做焦土和破门视觉，但不堵路。
3. 给人族防线做低矮防御道具和蓝色阵营识别。
4. 给主路增加车辙、弹坑贴片和烟尘，不新增阻挡。
5. 调整镜头目标和光照，让主战斗区更有纵深。
6. 给建筑、单位、特效做一次整体颜色平衡，避免某一种颜色过度抢焦点。
7. 做运行探针和截图验收，把场景问题记录到文档。
