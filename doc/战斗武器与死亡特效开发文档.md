# 战斗武器与死亡特效开发文档

更新时间：2026-05-28

本文记录坦克炮弹、士兵冲锋枪、飞机投弹、怪物锤击/踩踏，以及单位被击毁后的视觉与声音实现方案。目标是让后续开发能按同一套链路接特效，避免每个兵种各写一套临时逻辑。

## 1. 当前项目基础

当前项目已有这些基础能力：

- `Assets/Scripts/Effects/BattleEffectId.cs`：战斗特效枚举。
- `Assets/Scripts/Effects/EffectConfig.cs`：特效配置，绑定 prefab、对象池数量、音效。
- `Assets/Scripts/Effects/EffectManager.cs`：特效对象池和 fallback 粒子。
- `Assets/Scripts/Effects/PooledParticleEffect.cs`：粒子播放完自动回池。
- `Assets/Scripts/Audio/BattleAudioManager.cs`：音频对象池和播放限频。
- `Assets/Scripts/ApocalypseKingUnityGame.cs`：已有 `ProjectileView`、`SpawnProjectile`、`ResolveProjectileImpact`、`DeactivateHumanUnit`、`DefeatGiant`。

现状问题：

- `SpawnEffect` 仍是代码生成的球体火球/烟雾，表现力有限。
- 坦克已能从 `TankMuzzlePoint` 计算炮口点，但炮口火焰仍走临时 `SpawnEffect`。
- 士兵子弹目前是 LineRenderer 弹道，没有枪口火、弹壳、命中火花分层。
- 飞机目前使用 `Rocket` 抽象，需要改成“从弹舱落下的炸弹”。
- 怪物近战只有命中点火球，需要拆成锤击冲击、踩踏尘土、地裂、震屏。
- 单位死亡时直接 `SetActive(false)`，死亡动画、残骸、烟柱、爆炸序列不够完整。

## 2. 统一表现链路

所有武器都按同一条链路开发：

```text
攻击逻辑触发
  -> 计算发射点/命中点
  -> 播放开火特效和音效
  -> 生成弹体或射线
  -> 弹体飞行/射线命中
  -> 结算伤害
  -> 播放命中特效、爆炸、冲击波、屏幕震动
  -> 如果目标死亡，播放死亡特效和残骸表现
```

关键规则：

- 伤害永远由代码结算，不交给粒子碰撞。
- 粒子碰撞只负责火花、碎石、尘土二次反馈。
- 所有特效通过 `EffectManager.Play(...)` 播放，避免战斗中频繁 Instantiate/Destroy。
- 枪口火焰、炮口火焰应挂在发射点附近，但粒子主体用 World Simulation，避免坦克转向后火焰被拖走。
- 弹体、拖尾、命中爆炸分别做对象池；不要把所有表现塞进一个超大 prefab。

## 3. 推荐特效枚举

在 `BattleEffectId` 上扩展为更细的表现层：

```csharp
public enum BattleEffectId
{
    None = 0,

    MuzzleRifle,
    MuzzleTank,
    MuzzleAircraft,

    BulletTracer,
    BulletHitMetal,
    BulletHitDirt,

    ShellLaunchSmoke,
    ShellExplosionSmall,
    ShellExplosionLarge,

    BombDropTrail,
    BombExplosion,
    AirCrashExplosion,

    MonsterHammerImpact,
    MonsterStompDust,
    MonsterShockwave,
    ClawHit,

    SoldierDeath,
    TankDeathExplosion,
    TankWreckSmoke,
    AircraftDeathExplosion,
    AircraftCrashSmoke,
    MonsterDeathExplosion,
    MonsterDeathDust,

    HumanSummon,
    OrcSummon,
    HumanAirStrikeWarning,

```

第一版可以不一次性做全，但命名要先稳定。已有 `ExplosionSmall`、`ExplosionLarge` 可以保留为兼容入口，内部逐步替换到更明确的 `ShellExplosionSmall`、`BombExplosion`、`TankDeathExplosion`。

## 4. 发射点与骨骼挂点

需要在 `BattleUnit` 中增加统一挂点字段：

```csharp
public Transform muzzlePoint;
public Transform bombBayPoint;
public Transform leftFootPoint;
public Transform rightFootPoint;
public Transform hammerPoint;
public Transform deathEffectPoint;
```

挂点来源优先级：

1. 模型骨骼或节点名称匹配，例如 `Muzzle`、`Barrel`、`Gun`、`BombBay`、`Foot_L`、`Foot_R`、`Hand_R`。
2. 模型导入后自动创建 fallback 子节点，例如坦克炮口用当前 `TankMuzzlePoint` 的逻辑坐标。
3. 如果模型没有明确部件，使用单位中心加偏移，但要把偏移集中写在方法里，不散落在攻击逻辑中。

建议新增工具方法：

```csharp
private Transform FindChildByKeywords(Transform root, params string[] keywords)
private Transform CreateHardpoint(Transform parent, string name, Vector3 localPosition)
private Vector3 GetUnitHardpointWorld(BattleUnit unit, UnitHardpointKind kind)
```

注意：

- Quaternius/GLTF 模型节点名可能和预期不一致，所以必须有 fallback。
- 坦克炮塔已经有 `tankAimRoot`、`tankMuzzleVisual`，优先把 `muzzlePoint` 指向真实炮口或辅助炮口节点。
- 怪物锤击如果没有武器骨骼，就用右手/前爪附近 fallback；踩踏用脚底点。

## 5. 坦克炮弹

### 5.1 视觉目标

坦克攻击时需要看到：

- 炮塔朝向目标。
- 炮管口瞬间喷火。
- 炮口烟雾向前冲出。
- 炮管后坐。
- 炮弹从炮管飞出，有短拖尾。
- 命中怪物或地面时产生爆炸、火光、烟尘和冲击波。

### 5.2 代码接入点

当前入口是：

```text
FireHumanWeapon(...)
  -> unit.kind == UnitKind.Tank
  -> TankMuzzlePoint(unit)
  -> SpawnProjectile(ProjectileKind.Shell, ...)
```

改造建议：

1. 保留 `TankMuzzlePoint(unit)` 作为逻辑 fallback。
2. 新增 `GetTankMuzzleWorldPoint(unit)`，优先使用 `unit.muzzlePoint.position`。
3. 开火时播放：
   - `BattleEffectId.MuzzleTank`
   - `BattleEffectId.ShellLaunchSmoke`
   - `BattleAudioCueId.TankShot`
4. 弹体命中时播放：
   - `BattleEffectId.ShellExplosionLarge`
   - `BattleAudioCueId.ExplosionLarge`
   - 轻微 camera shake
5. 目标是怪物时调用 `DamageGiantAt` 或 `DamageGiantsInArea`。

伪代码：

```csharp
Vector3 muzzleWorld = GetTankMuzzleWorldPoint(unit);
Quaternion fireRotation = Quaternion.LookRotation(GetAimDirectionWorld(unit, target), Vector3.up);

EffectManager.Instance.Play(EffectPlayback.Create(
    BattleEffectId.MuzzleTank,
    muzzleWorld,
    fireRotation,
    null,
    1.2f));

BattleAudioManager.Instance.Play(BattleAudioCueId.TankShot, muzzleWorld);

SpawnProjectile(
    ProjectileKind.Shell,
    ProjectileTarget.Giant,
    fromX,
    fromZ,
    fromHeight,
    toX,
    toZ,
    toHeight,
    unit.damage,
    52f,
    520f,
    shellColor);
```

### 5.3 Prefab 参数

`FX_Muzzle_Tank`：

| 模块 | 建议 |
| --- | --- |
| Flash | 0.04 到 0.08 秒，橙白色，Cone shape，向前喷 |
| Smoke | 0.3 到 0.7 秒，灰黑色，World simulation |
| Sparks | 6 到 12 个，短生命周期，可不开碰撞 |
| Light | Point Light，0.06 到 0.12 秒后淡出 |

`FX_Shell_Explosion_Large`：

| 模块 | 建议 |
| --- | --- |
| Fireball | 0.18 到 0.35 秒，快速膨胀 |
| SmokeColumn | 1.2 到 2.0 秒，灰黑色上升 |
| DirtRing | 水平环形尘土，贴地扩散 |
| Debris | 8 到 18 个碎片，可开启低质量粒子碰撞 |
| Shockwave | 扁平透明圆环，0.2 秒内扩散并淡出 |

## 6. 士兵冲锋枪

### 6.1 视觉目标

士兵攻击时需要看到：

- 枪口短促闪光。
- 连续射击但不要每发都巨大火光。
- 子弹曳光线从枪口指向怪物。
- 命中怪物产生小火花/血雾/尘点。
- 枪声音频有限频，避免几十个士兵同时开火爆音。

### 6.2 代码接入点

当前入口是：

```text
FireHumanWeapon(...)
  -> unit.kind == UnitKind.Soldier
  -> SpawnProjectile(ProjectileKind.Bullet, ...)
```

建议：

1. 士兵使用 `muzzlePoint`，没有时使用 `unit.x + aim * 20f`。
2. 子弹可以继续使用 `ProjectileKind.Bullet`，但将 LineRenderer 拖尾材质统一复用。
3. 开火时播放 `MuzzleRifle`，概率播放弹壳粒子。
4. 命中怪物时播放 `BulletHitMetal` 或新增 `BulletHitFlesh`。
5. 枪声使用 `BattleAudioCueId.RifleShot`，`minInterval` 建议 0.045 到 0.08 秒。

### 6.3 连发节奏

现在士兵每次 `FireHumanWeapon` 发一发。冲锋枪表现可以有两种做法：

- 逻辑仍一发一结算，视觉用 `MuzzleRifle` 做 2 到 3 个短 burst。
- 逻辑一次攻击拆成 3 发微型子弹，每发伤害为 `unit.damage / 3`，间隔 0.05 秒。

第一版推荐前者，性能更稳。

### 6.4 Prefab 参数

`FX_Muzzle_Rifle`：

| 模块 | 建议 |
| --- | --- |
| Flash | 1 到 3 个粒子，生命周期 0.035 秒 |
| Smoke | 2 到 5 个粒子，生命周期 0.25 秒 |
| Spark | 可选，向前 3 到 6 个 |
| Light | 可选，小范围，持续 0.03 秒 |

`FX_BulletHit_Metal`：

| 模块 | 建议 |
| --- | --- |
| Spark Burst | 6 到 12 个，速度快，生命周期短 |
| Dust Puff | 小灰尘，0.2 到 0.4 秒 |
| Decal | 可选，命中点焦痕，池化并 8 到 12 秒回收 |

## 7. 飞机投弹

### 7.1 视觉目标

飞机攻击时不再像火箭直射，而是：

- 飞机飞到怪物上方或侧上方。
- 从机腹/弹舱释放炸弹。
- 炸弹受重力弧线下落，带轻微烟尾。
- 落地/命中后产生更大的地面爆炸。
- 如果飞机被击毁，空中爆炸后拖烟坠毁或直接碎片消失。

### 7.2 新增弹体类型

在 `ProjectileKind` 中增加：

```csharp
Bomb,
```

`UpdateProjectiles` 中将 `Bomb` 的轨迹改成更明显的下落弧线：

```csharp
bool gravityArc = shot.kind == ProjectileKind.Bomb;
float arc = gravityArc
    ? Mathf.Sin(t * Mathf.PI) * 0.35f - t * t * 1.6f
    : existingArc;
```

更稳定的方式是给 `ProjectileView` 增加 `velocityWorld`，用简化抛物线更新。第一版可以继续使用 from/to 插值加高度曲线。

### 7.3 代码接入点

当前飞机走 `FireHumanWeapon` 的最后分支，使用 `ProjectileKind.Rocket`。改为：

```text
unit.kind == UnitKind.Aircraft
  -> GetAircraftBombBayPoint(unit)
  -> 播放 BombDropTrail 或释放烟粒
  -> SpawnProjectile(ProjectileKind.Bomb, ...)
```

命中时：

```text
ResolveProjectileImpact(...)
  -> ProjectileKind.Bomb
  -> Play BombExplosion
  -> DamageGiantsInArea(...)
  -> camera shake
```

### 7.4 Prefab 参数

`FX_BombDropTrail`：

- 少量灰白烟，跟随炸弹或沿途生成。
- 生命周期 0.4 到 0.8 秒。
- 不开启碰撞。

`FX_BombExplosion`：

- 规模大于炮弹爆炸。
- 火球更宽，尘环更大。
- 可以额外产生 2 到 3 个延迟小爆点，增强空袭感。

## 8. 怪物锤击和踩踏

### 8.1 视觉目标

怪物攻击要区分目标类型：

- 打坦克：锤击/重拳冲击，金属火花、碎片、强震屏。
- 踩士兵：脚底尘土、地裂、冲击波，士兵死亡烟尘较小。
- 打飞机：上挥/拍击，空中火花或爆点。

### 8.2 代码接入点

当前入口是：

```text
PerformGiantMeleeAttack(...)
  -> SpawnEffect(impactX, impactZ, ...)
  -> ApplyGiantContactDamage(...)
```

改造建议：

```csharp
BattleEffectId impactEffect = target.kind == UnitKind.Tank
    ? BattleEffectId.MonsterHammerImpact
    : target.kind == UnitKind.Aircraft
        ? BattleEffectId.ClawHit
        : BattleEffectId.MonsterStompDust;

EffectManager.Instance.Play(EffectPlayback.Create(
    impactEffect,
    ToWorldPoint(impactX, impactZ, target.kind == UnitKind.Aircraft ? 2.5f : 0.08f),
    Quaternion.identity,
    null,
    target.kind == UnitKind.Tank ? 1.6f : 1.15f));
```

再叠加：

- `MonsterShockwave`：只在打坦克或踩踏多人时触发。
- `BattleAudioCueId.CreatureHit` 或新增 `MonsterStomp`。
- Camera shake：踩踏 0.10 到 0.18 秒，锤击 0.18 到 0.28 秒。

### 8.3 动画事件

如果怪物动画 clip 有明确的出拳/落脚时刻，最佳方案是添加 Animation Event：

```text
Animation Event: OnMonsterImpact
参数：leftFoot / rightFoot / hammer
```

第一版可以继续在 `PerformGiantMeleeAttack` 触发特效，但后续要迁移到动画事件。这样不会出现怪物手还没挥到，爆炸已经出来的错位感。

## 9. 单位死亡效果

### 9.1 士兵死亡

当前 `DeactivateHumanUnit` 直接隐藏单位。建议改成：

```text
HP <= 0
  -> active = false，停止参与战斗
  -> 不立刻 root.SetActive(false)
  -> 播放 Death 动画或倒地程序动画
  -> 播放 SoldierDeath 小尘土/火花
  -> 0.8 到 1.5 秒后隐藏并回收
```

如果士兵模型有 `Death` 动画：

- 设置 `unit.deathTimer`。
- 播放 `Death` clip。
- 死亡期间不再移动、不再攻击、不再被选为目标。

如果没有死亡动画：

- 模型向后/侧边倾倒 70 到 90 度。
- 透明淡出或直接延迟隐藏。

### 9.2 坦克击毁

坦克死亡建议分三层：

1. 第一帧：`TankDeathExplosion`，火球和冲击波。
2. 0.1 到 0.3 秒：炮塔/履带小火花，碎片飞出。
3. 2 到 6 秒：`TankWreckSmoke` 持续黑烟，保留低模残骸或烧焦底盘。

实现方式：

- 死亡时先生成一个 `Wreck` 物体，使用坦克当前 transform。
- 原单位 root 隐藏回池。
- Wreck 进入 `wrecks` 列表，生命周期到期后隐藏。

不要让死亡坦克继续占用 `tanks` 单位逻辑，也不要继续参与建筑避障。

### 9.3 飞机击毁

飞机死亡建议：

- 空中触发 `AircraftDeathExplosion`。
- 播放短暂火球和碎片。
- 如果要增强表现，生成一个坠落残骸，沿当前方向下落 0.6 到 1.2 秒。
- 落地时播放 `AircraftCrashSmoke` 或 `BombExplosion` 的小版本。

第一版可以只做空中爆炸和烟柱，避免坠落残骸穿建筑。

### 9.4 怪物死亡

怪物死亡建议：

1. 停止攻击和移动。
2. 播放 `Death` 动画或程序化向后倒下。
3. 身体周围连续播放 2 到 3 个爆点。
4. 落地瞬间播放 `MonsterDeathDust` 大尘环和震屏。
5. 1.5 到 3 秒后隐藏。

当前 `DefeatGiant` 是立即隐藏并生成几个临时火球。后续应改为死亡状态：

```csharp
giant.active = false;
giant.dying = true;
giant.deathTimer = 2.2f;
PlayDeathAnimation(giant);
```

并在 `UpdateDeathVisuals(dt)` 中处理倒地、爆点和最终回收。

## 10. 建议新增运行时结构

### 10.1 ProjectileSpec

将当前 `SpawnProjectile` 的长参数收敛成结构体：

```csharp
private struct ProjectileSpec
{
    public ProjectileKind kind;
    public ProjectileTarget target;
    public Vector3 fromWorld;
    public Vector3 toWorld;
    public float damage;
    public float radius;
    public float speed;
    public Color color;
    public BattleEffectId trailEffect;
    public BattleEffectId impactEffect;
    public BattleAudioCueId impactAudio;
}
```

好处：

- 坦克炮弹、炸弹、石头、子弹可以共享一套接口。
- 命中特效不需要在 `ResolveProjectileImpact` 里写大量 `if`。
- 后续给弹体加拖尾、光源、旋转也更清楚。

### 10.2 DeathVisual

新增死亡表现池：

```csharp
private sealed class DeathVisual
{
    public UnitKind kind;
    public GameObject root;
    public float life;
    public float maxLife;
    public Vector3 velocity;
    public bool active;
}
```

用于坦克残骸、飞机坠落、怪物倒地临时对象。这样单位对象可以立即退出战斗逻辑，视觉仍能保留。

### 10.3 Camera Shake

新增轻量震屏：

```csharp
private float cameraShakeTime;
private float cameraShakeAmplitude;
```

触发规则：

| 事件 | 时长 | 幅度 |
| --- | --- | --- |
| 士兵死亡 | 0 | 0 |
| 子弹命中 | 0 | 0 |
| 炮弹爆炸 | 0.08 到 0.14 | 0.04 到 0.08 |
| 炸弹爆炸 | 0.16 到 0.24 | 0.09 到 0.16 |
| 怪物踩踏 | 0.12 到 0.20 | 0.08 到 0.14 |
| 坦克击毁 | 0.18 到 0.28 | 0.12 到 0.20 |
| 怪物死亡 | 0.25 到 0.38 | 0.18 到 0.28 |

竖屏直播画面中震屏一定要克制，避免观众看不清弹幕和单位。

## 11. 推荐开发顺序

### 第一阶段：替换临时特效

目标：不改大结构，先让效果明显提升。

1. 扩展 `BattleEffectId`。
2. 为 `MuzzleRifle`、`MuzzleTank`、`ShellExplosionLarge`、`BombExplosion`、`MonsterStompDust`、`TankDeathExplosion` 建 prefab。
3. 在 `FireHumanWeapon` 中把开火 `SpawnEffect` 替换为 `EffectManager.Play`。
4. 在 `ResolveProjectileImpact` 中把爆炸 `SpawnEffect` 替换为 `EffectManager.Play`。
5. 在 `DeactivateHumanUnit`、`DefeatGiant` 中接入死亡特效。

### 第二阶段：飞机投弹

目标：把飞机从直射火箭改成投弹。

1. 新增 `ProjectileKind.Bomb`。
2. 新增 `bombBayPoint` 或 fallback 机腹发射点。
3. 修改飞机攻击分支，生成 Bomb。
4. Bomb 命中时范围伤害和大爆炸。
5. 探针检查炸弹轨迹不会穿过地面后继续飞。

### 第三阶段：死亡状态

目标：单位死亡后有动画或残骸，不再瞬间消失。

1. 为 `BattleUnit` 增加 `dying`、`deathTimer`。
2. 死亡时停止参与战斗目标选择。
3. 士兵/怪物播放 Death 动画或程序化倒地。
4. 坦克/飞机生成残骸视觉对象。
5. 死亡视觉到时回收。

### 第四阶段：动画事件与精修

目标：攻击动作和特效时机准确。

1. 给士兵 Shoot 动画加枪口事件。
2. 给怪物 Punch/Stomp 动画加 impact 事件。
3. 炮口后坐与 `MuzzleTank` 同步。
4. 给大爆炸加入 decal、地裂、烟柱和音频低频层。

## 12. 性能预算

竖屏直播和大量单位场景中，建议预算：

| 类型 | 同屏上限 | 说明 |
| --- | --- | --- |
| Bullet tracer | 80 到 120 | 生命周期短，LineRenderer 池化 |
| MuzzleRifle | 40 到 64 | 小粒子，无碰撞 |
| MuzzleTank | 12 到 20 | 中粒子，无碰撞 |
| Large explosion | 8 到 12 | 大粒子，少量碎片碰撞 |
| Smoke column | 12 到 20 | 生命周期长，要限量 |
| Wreck visuals | 12 到 24 | 超过上限直接回收最旧 |
| AudioSource | 小于 48 | 沿用 `BattleAudioManager.maxSources` |

优化规则：

- 同类大爆炸短时间内超过上限时，只播放伤害和声音，跳过部分视觉层。
- 远处士兵枪口火可以按概率播放，例如 40%。
- 大量子弹不要每发生成独立 Point Light。
- 拖尾材质、爆炸材质、烟雾材质必须复用，不在运行时频繁 new Material。

## 13. 验收清单

坦克：

- 炮弹从炮管口附近出现，而不是从车身中心出现。
- 炮口火焰方向和炮塔朝向一致。
- 炮弹飞行轨迹能看到拖尾。
- 命中怪物时出现炮弹爆炸和烟尘。
- 坦克死亡时有大爆炸、黑烟或残骸。

士兵：

- 士兵攻击时枪口有短促闪光。
- 子弹曳光从枪口射向目标。
- 枪声不会因多名士兵同时射击而爆音。
- 士兵死亡不是无声瞬间消失，至少有小尘土/倒地/淡出。

飞机：

- 炸弹从机腹或机身下方释放。
- 炸弹轨迹是下落弧线，不是水平火箭。
- 爆炸发生在地面/目标附近。
- 飞机被击毁时在空中爆炸或冒烟坠落。

怪物：

- 锤击坦克和踩踏士兵使用不同特效。
- 冲击波贴地扩散，不漂浮到空中。
- 怪物攻击特效和动画动作时机接近。
- 怪物死亡有大尘土/爆点/震屏，不是直接消失。

通用：

- 所有新特效都通过 `EffectManager` 池化。
- 所有新音效都通过 `BattleAudioManager` 播放。
- 粒子碰撞不负责伤害。
- 构建后无 C# 编译错误。
- 探针或手动观察确认帧率没有明显下降。

## 14. 常见坑

- 不要把爆炸伤害写在粒子碰撞回调里；粒子数量、帧率和碰撞质量会导致伤害不稳定。
- 不要死亡时先隐藏单位再读取挂点；需要先缓存死亡位置和朝向。
- 不要让烟柱 prefab 永久循环；必须能自动停止并由 `PooledParticleEffect` 回池。
- 不要把枪口火挂在会快速旋转的子节点上并使用 Local Simulation，否则火焰会被炮塔运动拖歪。
- 不要每个子弹都播放 3D 枪声；士兵多时必须限频或合并音频。
- 不要让飞机炸弹继续使用 `Rocket` 逻辑，否则视觉方向会和“投弹”预期冲突。
- 不要只在编辑器里看 prefab；必须在竖屏运行视角里看比例，很多特效近看漂亮、远看只剩一团亮斑。
- 不要把死亡残骸继续当单位参与寻敌和避障；残骸只属于视觉层。
