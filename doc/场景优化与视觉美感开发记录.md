# 场景优化与视觉美感开发记录

更新时间：2026-05-28

## 目标

把当前直播弹幕战场从“能跑、能看”推进到“性能稳定、布局合理、镜头有层次、战斗重点清楚”。本项目的场景大部分由 `ApocalypseKingUnityGame` 在运行时生成，因此优化方案要以代码落地为主，不能只依赖 Unity Inspector 里的手工勾选。

## 当前项目特点

- 主场景由 `Assets/Scripts/ApocalypseKingUnityGame.cs` 运行时创建。
- 中世纪村庄使用 Quaternius `MedievalVillageMegaKit` 模块组合，不是手工摆放的整场景 Prefab。
- 单位移动不是 Rigidbody/Collider 驱动，而是使用 `unit.x` / `unit.z` 逻辑坐标和数学避障。
- 炮弹、短生命周期特效、死亡残骸已经有池化雏形，后续应继续预热和复用。
- HUD 目前集中在一个 Canvas 下，高频文本和静态背景混在一起，后续要拆分动静层。

## 落地原则

1. 先稳性能，再堆视觉效果。
2. 先解决可读性，再追求华丽。
3. 用代码记录场景规则，避免运行时生成场景和手工 Inspector 配置脱节。
4. 建筑可见模型和建筑阻挡数据必须同源创建，新增建筑时同步 `AddBuildingObstacle()`。
5. 临时截图、日志和探针文件用完即删，不进入提交。

## 技术优化计划

### 1. HUD 动静分离

现状：顶部战况、底部日志、分辨率按钮、血条和静态背景都在同一个 HUD Canvas 中。

改造：
- 静态面板、标题、背景框放到静态 Canvas。
- 分数、倒计时、血条、日志、Banner、Loading 放到动态 Canvas。
- 分辨率按钮作为调试控件保留，但默认隐藏，避免最终画面被调试 UI 抢焦点。

验收：
- 高频 `RefreshHud()` 只影响动态 Canvas 下的元素。
- 安全区适配仍然正常。
- 分辨率调试按钮可通过代码开关恢复。

### 2. 对象池预热

现状：`MaxProjectiles`、`MaxEffects`、`MaxDeathVisuals` 已经限制最大数量，但部分对象会在第一次使用时创建。

改造：
- 模型加载完成后预热常用炮弹、子弹、爆炸、死亡残骸对象。
- 保持 `SetActive(true/false)` 循环复用，避免战斗高潮时出现 Instantiate 峰值。

验收：
- 开战后第一次开火、第一次爆炸、第一次死亡没有明显卡顿。
- 不引入额外 Destroy。

### 3. 材质复用与 GPU Instancing

现状：运行时材质有缓存，但重复模型和村庄模块还可以更明确地开启实例化。

改造：
- 对运行时生成的 opaque/transparent 材质开启 `enableInstancing`。
- 对 Quaternius 村庄模块、坦克变体、重复道具尽量复用 sharedMaterial。
- 骨骼动画单位不强求 instancing，优先保证动画正常。

验收：
- Draw Calls 降低或不升高。
- 不出现粉色材质、透明排序异常或贴图丢失。

### 4. 建筑与道路避障

现状：建筑阻挡已经使用逻辑坐标 AABB，单位能避免直接穿建筑，但队形容易显得被硬推开。

改造：
- 把村庄主路、广场、入口道路做成明确的“通行带”。
- 单位绕建筑时优先贴近道路方向，而不是只按建筑上下侧绕行。
- 坦克和怪物使用更大的避障半径，士兵允许更细的队列间距。

验收：
- `buildingOverlaps` 保持为 0。
- 坦克不会挤成一坨。
- 怪物能从入口压进村庄，而不是卡在建筑边。

## 场景美术计划

### 1. 战场叙事

场景按“村庄被卷入弹幕战争”组织：
- 左侧或近景是人族集结区。
- 中央是村庄主街和广场。
- 右侧或远景是怪物入口。
- 教堂/塔楼作为视觉焦点，酒馆、谷仓、民居围出空间。

### 2. 视觉层级

- 主路形成对角线，帮助镜头产生纵深。
- 高建筑只放在边缘和焦点处，不遮挡主要战斗。
- 小道具只放在路边，避免影响单位可读性。
- 爆炸、枪口火光和死亡特效按强度分级：普通命中小，坦克炮命中中，死亡爆炸大。

### 3. 光照与色彩

- 保持内置管线，暂不迁移 URP/HDRP，降低材质兼容风险。
- 用方向光、环境光、雾色和 Blob Shadow 提升立体感。
- 人族保留冷蓝提示，怪物侧保留暖红提示，村庄使用木色、石色和草绿。
- 避免高饱和紫色成为错误视觉焦点。

### 4. 镜头

- 保持直播竖屏可读性。
- 默认镜头调整为 3/4 斜俯视，让主路和队形更有纵深。
- 相机抖动只用于坦克炮、炸弹、怪物重击和死亡爆炸，避免普通枪击抖动过多。

## 开发顺序

1. HUD 动静分离与调试按钮默认隐藏。
2. 运行时材质 instancing 和装饰物渲染设置整理。
3. 村庄布局重排：主街、广场、入口、视觉焦点。
4. 建筑绕行从“避开矩形”升级为“贴近道路绕行”。
5. 对象池预热和战斗特效强弱分层。
6. 运行探针、截图检查、临时文件清理、中文提交并 push。

## 容易踩坑

- 不要把 Inspector 里的 Static 勾选当成唯一方案，本项目大量物体运行时创建。
- 不要给复杂建筑加 MeshCollider，本项目移动不是物理驱动。
- 不要让建筑阻挡过密，坦克和怪物半径大，会被窄路卡住。
- 不要把所有爆炸都做成同样大小，视觉重点会乱。
- 不要直接迁移 URP/HDRP，除非先确认所有 glTF、粒子和运行时材质都能正常渲染。
