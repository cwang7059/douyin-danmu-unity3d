# 中世纪村庄与建筑避障实现记录

更新时间：2026-05-27

## 目标

把战场背景从现代废墟改成中世纪村庄，并保证士兵、坦克、怪物和飞机不会直接穿过房屋、塔楼、水井等建筑。

## 实现位置

- 主逻辑：`Assets/Scripts/ApocalypseKingUnityGame.cs`
- 运行探针：`Assets/Scripts/ApocalypseKingRuntimeProbe.cs`

## 坐标约定

项目单位移动使用自己的逻辑坐标：

- `unit.x` / `unit.z` 是战斗逻辑坐标。
- `ToWorldPoint(x, z, height)` 负责把逻辑坐标乘以 `LogicalToWorld` 转成 Unity 世界坐标。
- 建筑阻挡必须记录在逻辑坐标里，不能只依赖 Unity Collider，否则单位视觉模型会绕不过去，因为移动不是由物理系统驱动的。

## 村庄渲染

村庄建筑由运行时 primitive 拼出来：

- `CreateMedievalVillage()` 创建房屋、马厩、谷仓、教堂塔楼、水井、市场摊位和木栅栏。
- `CreateVillagePaths()` 创建泥路、岔路和石块路面。
- `CreateVillageLandingFields()` 用木板和草地替换现代停机坪。

房屋和塔楼创建时会同步调用：

```csharp
AddBuildingObstacle(name, centerX, centerZ, halfX, halfZ, height, padding);
```

这样“看见的建筑”和“实际阻挡区”来自同一次创建流程，避免后续改场景时忘了改碰撞。

## 避障规则

建筑阻挡区使用逻辑坐标 AABB：

- `CenterX` / `CenterZ`：建筑中心。
- `HalfX` / `HalfZ`：建筑半宽和半深。
- `Padding`：视觉模型外的安全余量。
- 单位自身会再加一层半径：士兵、坦克、飞机、怪物各不相同。

移动时走两步：

1. `ApplyBuildingAvoidanceSteering()` 检查当前点到目标点的线段是否会撞到建筑扩展 AABB；如果会，先把目标 `z` 偏到建筑上方或下方，形成绕行倾向。
2. `MoveUnitToAvoidingBuildings()` 把移动拆成小步，每一步后调用 `ClampUnitPosition()`，再由 `ResolveBuildingCollision()` 把已经进入阻挡区的单位推出去。

这比只在移动结束时检测更稳，能减少高速单位一帧跨进建筑深处的情况。

## 覆盖单位

这些单位会避开建筑：

- `Soldier`
- `Tank`
- `Aircraft`
- `Giant`

弹丸和特效不参与建筑避障，目前仍可穿过建筑。

## 验证指标

运行 `ApocalypseKingRuntimeProbe` 后关注日志：

- `buildings`：当前注册的建筑阻挡区数量。
- `buildingOverlaps`：当前仍压在建筑阻挡区里的活跃单位数量，应为 `0`。
- 同时确认截图中建筑和单位没有明显重叠。

## 后续容易踩坑

- 新增建筑时不要只创建模型，一定要同步 `AddBuildingObstacle()`。
- 不要用 Unity 物理 Collider 作为唯一碰撞来源，本项目单位不是 Rigidbody 驱动。
- 建筑阻挡不要太密，怪物和坦克半径大，连续建筑会把它们卡在路口。
- 如果建筑有大角度旋转，当前阻挡仍是 AABB；视觉上可以小角度旋转，大角度建筑需要单独扩大 `Padding`。
- 飞机也走同一套平面避障；如果以后希望飞机“飞越低矮建筑”，需要给 `BuildingObstacle.Height` 加高度判定。
