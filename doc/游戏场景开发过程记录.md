# 游戏场景开发过程记录

更新时间：2026-05-28

本文记录“草地、远山、落日、海岸边”场景替换的实际开发过程，便于后续继续排查和迭代。

## 2026-05-28 场景素材替换

### 目标

- 替换现有厚方块地面，改成草地质感的战场地面。
- 去掉远处突兀的矩形方块山脊，改成远山轮廓。
- 增加落日氛围、海面、海岸沙地。
- 保留当前中世纪村庄、建筑阻挡、道路通行带和单位避障逻辑。

- `Assets/DownloadedAssets/Environment/Online/PolyHavenCoastalSunset`

### 实现计划

1. 新增运行时环境贴图资源常量。
2. 在 `CreateCoreScene()` 中切换为落日天空盒、暖色方向光和更柔和的雾色。
3. 把 `CreateGround()` 从厚 Cube 改为贴图 Plane，减少“地面大方块”观感。
4. 把 `CreateTerrainDepth()` 的远景方块替换为：
   - 海面 Plane
   - 海岸沙地 Plane
   - 半透明浅水/反光 Plane
   - 远山 Mesh 轮廓
   - 落日太阳盘和地平线光晕
5. 把主路/广场从厚 Cube 改为贴地 Plane，继续保留 `AddRoadCorridor()`。
6. Unity 构建验证，清理临时日志。

## 2026-05-28 直升机尺寸与旋翼修复

### 问题

- KumaSousa 直升机 FBX 导入后源对象根节点带 `100` 倍缩放，且主旋翼直径远大于机身高度；继续用原 `1.18` 目标高度会导致视觉上明显偏大。
- 模型真实旋翼节点不是 `Rotor/Propeller/Blade`，而是主旋翼 `Heli`、尾旋翼 `Tail`；只按通用关键词查找会绑不到真实旋翼，最终只看到静态模型或很小的辅助旋翼。

### 修复

- 将 `Aircraft` 模型目标高度调整为 `0.62`，缩小机身和旋翼占屏比例。
- 直升机不再依赖 Animator 播放 FBX 旋翼动画，而是运行时直接绑定 `Heli` 和 `Tail` 节点并强制持续旋转。
- 保留通用 `Rotor/Propeller/Blade` 匹配和透明辅助旋翼兜底，避免以后替换其他直升机模型时再次失效。
- 已用 Unity 导入检查确认该 FBX 动画片段包含 `Spinning/Start_Spinning/Stop_Spinning`，层级包含 `Heli/Tail`；检查日志用完已删除。

### 后续避坑

- 直升机尺寸不要只看模型高度，还要看旋翼直径；旋翼直径通常才是屏幕占用的主要原因。
- 换飞行器模型后必须检查实际节点名，不要默认资产会叫 `Rotor`。
- 如果 Animator 动画和代码旋翼同时控制同一节点，可能互相覆盖；本项目直升机旋翼统一由代码控制更稳定。

## 2026-05-28 直升机朝向、投弹与建筑破坏

### 目标

- 直升机不再固定朝一个方向，而是持续朝向最近的怪物。
- 直升机保留投掷炸弹能力，投弹点从机身前方生成，落点直接瞄准怪物。
- 坦克炮弹/火箭遇到建筑时不再穿过去，而是在建筑位置爆炸。
- 炸弹、炮弹、火箭的爆炸范围可以扣除建筑血量，建筑被打掉后从画面和避障系统里同步移除。

### 实现记录

- 在 `UpdateHumanUnit()` 中为 `Aircraft` 增加 `UpdateAircraftAiming()`，用 `headingDegrees` 和 `turretYawDegrees` 平滑转向最近怪物。
- `UsesDynamicHeading()` 现在包含 `Aircraft`，模型渲染会跟随 `headingDegrees` 旋转。
- `FireHumanWeapon()` 的直升机分支把炸弹出生点前移到机头方向，并将落点改为怪物当前位置。
- `BuildingObstacle` 新增 `Root`、`Hp`、`Destroyed`，MegaKit 建筑和 fallback 建筑创建时都把可见根节点绑定到障碍数据。
- `UpdateProjectiles()` 对炮弹/火箭做飞行段与建筑 AABB 的相交检测，先撞到建筑就提前结算爆炸。
- `ResolveProjectileImpact()` 对炮弹/火箭/炸弹调用 `DamageBuildingsInArea()`；建筑血量归零后隐藏原建筑、生成低矮废墟，并播放大爆炸和烟雾。
- 建筑销毁后所有避障检测会跳过 `Destroyed`，避免出现“建筑消失但单位仍绕行/被空气墙挡住”的问题。

### 后续避坑

- 可见建筑根节点必须和 `BuildingObstacle` 绑定，否则只能炸掉数据或只能隐藏画面，二者会脱节。
- 炮弹飞行命中建筑用的是逻辑坐标 AABB，不要改成 Unity Collider；本项目单位与弹丸移动都不是物理系统驱动。
- 炸毁建筑不要从 `buildingObstacles` 列表里移除，直接标记 `Destroyed` 更安全；运行时遍历不会因为索引变化出错。
- 如果后续要让飞机飞越低矮建筑，需要基于 `BuildingObstacle.Height` 区分地面单位和空中单位，现在仍按平面障碍处理。
