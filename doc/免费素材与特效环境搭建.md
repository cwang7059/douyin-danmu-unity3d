# 免费素材地址与特效环境搭建清单

更新时间：2026-05-27

本文件给开发人员直接使用：到哪里拿素材、拿什么、放到工程哪里、导入后如何搭建 Unity3D 的特效、声音和粒子碰撞环境。

说明：

- 下面地址优先选择官方站点、Unity 官方商城页面、CC0 或明确 FREE 的资源。
- 第三方资源未来可能改价、下架或改许可证；正式商用前必须重新打开资源页核对 `Price / License / EULA`，并保存许可证截图或文本到 `doc/许可证/`。
- CC0 资源可以作为首选；Unity Asset Store 的 FREE 资源一般走 `Standard Unity Asset Store EULA`，不能把原始素材单独转卖。
- OpenGameArt、Poly Pizza、Freesound 每个资源许可证可能不同，只能下载许可证明确适合项目的条目。

## 2. 本项目首选资源组合

### 人族阵营

推荐起步：

- 基础步兵：当前项目已有 `soldier.glb`，也可从 `Poly Pizza` 搜索 `soldier`，只选 CC0 或记录 CC-BY 署名。
- 近战英雄/盾兵：`Meshtint Free Knight`。
- 平民/炮灰兵：`Lowpoly Medieval Peasants - Free`。
- 人形动作：`Quaternius Universal Animation Library` 或 `Mixamo`。
- UI/图标：`Kenney` 的 UI、icons、game assets。

落地要求：

- 人形模型导入后设置 `Rig > Animation Type = Humanoid`。
- 动作文件单独放到 `Assets/Animations/Human/`。
- 不直接改第三方 prefab，复制到 `Assets/Prefabs/Units/Human/` 后再做项目内 prefab。

### 兽族阵营

推荐起步：

- 小怪、飞行怪、恶魔、龙、地狱犬替代：`Quaternius Ultimate Monsters`。
- Boss 或精英怪：从 `Ultimate Monsters` 里挑体型更大的 Demon/Dragon/Skull 类模型。
- 怪物动作：优先使用资源自带动画；四足或特殊骨骼不要强行套 Humanoid。
- 怪物音效：Sonniss GDC、Pixabay、Freesound 中搜索 `monster roar`、`creature growl`、`dog beast`，只选可商用许可证。

落地要求：

- 兽族模型统一朝向：在 prefab 根节点下增加 `ModelRoot`，只在 `ModelRoot` 上修正 yaw。
- 每次换模型后用运行探针检查移动方向，不只看静态截图。
- 四足怪物不要使用人形动画 retarget，除非模型本身提供兼容骨架。

### 战场环境

推荐起步：

- 人族基地：`Quaternius Ultimate Fantasy RTS` 或 `Medieval Village MegaKit`。
- 森林/边界装饰：`Quaternius Ultimate Fantasy RTS`、`Poly Haven Models`。
- 地面贴图：`ambientCG` 搜索 `Ground`、`Dirt`、`Rock`、`Mud`。
- HDRI/天空光：`Poly Haven HDRIs`，低端机可只用 Directional Light，不启用实时全局光。

导入路径：

```text
Assets/Art/Environment/
  Battlefield/
  HumanBase/
  OrcBase/
  Props/
  Materials/
```

## 4. Unity 特效环境搭建

### 4.1 当前项目推荐方案

当前项目已经能加载 GLB，并且主要使用运行时代码构建场景。为了稳定，第一阶段建议：

- 继续使用 `ParticleSystem` 做主要特效。
- 暂时不强制切 URP。
- 从 Unity Asset Store 导入 `War FX` 和 `Cartoon FX Remaster Free`。
- 自己封装 `EffectManager` 和对象池。
- 粒子碰撞只做视觉反馈，不做核心伤害。

### 4.2 Built-in 管线特效基础设置

步骤：

1. 打开 Unity。
2. `Window > Package Manager > My Assets`。
3. 搜索并导入：
   - `War FX`
   - `Cartoon FX Remaster Free`
   - `Simple FX - Cartoon Particles`
4. 导入后不要直接使用原始 prefab，复制需要的 prefab 到：

```text
Assets/Prefabs/Effects/
  Explosion/
  Hit/
  Muzzle/
  Magic/
  Buff/
  Summon/
```

5. 把复制后的 prefab 命名成项目统一格式：

```text
FX_Explosion_Fire_Small
FX_Explosion_Fire_Large
FX_Hit_Bullet_Metal
FX_Hit_Claw_Blood
FX_Muzzle_Rifle
FX_Muzzle_Tank
FX_Skill_AirStrike_Warning
FX_Skill_OrcRage_Buff
FX_Summon_Human
FX_Summon_Orc
```

### 4.3 URP/VFX Graph 可选方案

只有在需要 GPU 大规模粒子时再启用 VFX Graph，例如魔法风暴、虫群、上千粒子的能量雨。

URP 切换步骤：

1. 先备份工程或建分支。
2. `Window > Package Manager > Unity Registry`。
3. 安装 `Universal RP`。
4. `Assets > Create > Rendering > URP Asset`。
5. `Project Settings > Graphics` 指向新的 URP Asset。
6. `Project Settings > Quality` 每个质量档也指向 URP Asset。
7. `Edit > Render Pipeline > Universal Render Pipeline > Upgrade Project Materials to URP Materials`。
8. 检查第三方粒子材质，必要时改为 `Universal Render Pipeline/Particles/Unlit` 或包自带 URP shader。

VFX Graph 安装步骤：

1. `Window > Package Manager > Unity Registry`。
2. 搜索 `Visual Effect Graph`。
3. 安装与 Unity/SRP 匹配的版本。
4. `Assets > Create > Visual Effects > Visual Effect Graph` 创建效果。
5. 场景中添加 `Visual Effect` 组件并绑定 graph asset。

注意：

- Unity 文档说明 VFX Graph 是基于 GPU/Compute Shader 的节点式视觉特效系统。
- 移动端和直播推流电脑要谨慎使用 VFX Graph，先用 `ParticleSystem` 完成基础版本。
- 当前大战类项目中，爆炸、命中、烟雾、枪口火光用 `ParticleSystem` 已足够。

## 5. Layer 与碰撞矩阵

建议新增 Layers：

```text
Unit_Human
Unit_Orc
Projectile_Human
Projectile_Orc
FX_Collision
FX_NoCollision
Ground
Obstacle
UI3D
```

`Project Settings > Physics` 碰撞矩阵建议：

| Layer | 与谁碰撞 |
| --- | --- |
| `Unit_Human` | `Unit_Orc`, `Obstacle` |
| `Unit_Orc` | `Unit_Human`, `Obstacle` |
| `Projectile_Human` | `Unit_Orc`, `Ground`, `Obstacle` |
| `Projectile_Orc` | `Unit_Human`, `Ground`, `Obstacle` |
| `FX_Collision` | `Ground`, `Obstacle` |
| `FX_NoCollision` | 不参与物理碰撞 |

原则：

- 子弹、炮弹、技能伤害使用代码判定：`Raycast`、`SphereCast`、`OverlapSphere`、`OverlapBox`。
- 粒子 Collision 只用于火花、碎石、烟尘贴地、能量粒子撞护盾。
- 不让每个粒子承担伤害逻辑。

## 6. 粒子碰撞配置

适用特效：

- `FX_Spark_Metal`：金属火花。
- `FX_Debris_Stone`：碎石落地。
- `FX_Blood_Spray`：血滴短距离喷溅。
- `FX_Magic_Shield_Sparks`：魔法粒子撞护盾。

推荐 Collision Module 参数：

```text
Type: World
Mode: 3D
Collides With: Ground, Obstacle, FX_Collision
Dampen: 0.35
Bounce: 0.15
Lifetime Loss: 0.35
Min Kill Speed: 0.1
Collision Quality: Medium 或 Low
Enable Dynamic Colliders: 关闭，除非明确需要
Send Collision Messages: 只在需要脚本回调时开启
Max Collision Shapes: 控制在 32 或更低
```

开发规则：

- 爆炸主体、烟雾主体不开 Collision。
- 火花和碎片可以开 Collision。
- 一屏超过 20 个同类特效时，自动切到无碰撞版本。
- 移动端配置默认关闭粒子碰撞。

## 7. EffectManager 搭建规范

目录：

```text
Assets/Scripts/Effects/
  EffectId.cs
  EffectConfig.cs
  EffectManager.cs
  PooledParticleEffect.cs
  ParticleCollisionRelay.cs
```

`EffectConfig` 建议字段：

```csharp
public enum EffectId
{
    ExplosionSmall,
    ExplosionLarge,
    BulletHitMetal,
    BulletHitDirt,
    ClawHit,
    MuzzleRifle,
    MuzzleTank,
    HumanSummon,
    OrcSummon,
    HumanAirStrikeWarning,
    OrcRageBuff
}
```

```csharp
[CreateAssetMenu(menuName = "Battle/Effect Config")]
public sealed class EffectConfig : ScriptableObject
{
    public EffectId id;
    public GameObject prefab;
    public int prewarmCount = 8;
    public int maxCount = 64;
    public bool attachToParent;
    public bool allowCollision;
    public AudioClip[] sounds;
}
```

播放流程：

```text
DamageSystem / SkillSystem
  -> EffectManager.Play(effectId, position, rotation)
  -> 从对象池取 prefab
  -> 设置位置、缩放、阵营颜色
  -> 播放 ParticleSystem
  -> 通知 AudioManager 播放音效
  -> 粒子结束后回池
```

## 8. 声音环境搭建

目录：

```text
Assets/Audio/
  BGM/
  SFX/
    Weapon/
    Explosion/
    Creature/
    Magic/
    UI/
  Voice/
  Mixer/
```

AudioMixer：

```text
BattleAudioMixer
  Master
    BGM
    SFX
      Weapon
      Explosion
      Creature
      Magic
      UI
    Voice
    Ambience
```

导入设置：

| 音频类型 | Load Type | Compression | Spatial Blend |
| --- | --- | --- | --- |
| UI 短音效 | Decompress On Load | Vorbis/PCM | 2D |
| 枪声/命中 | Compressed In Memory | Vorbis | 2D 或 3D |
| 爆炸/Boss | Compressed In Memory | Vorbis | 3D |
| BGM | Streaming | Vorbis | 2D |
| 环境声 | Streaming 或 Compressed | Vorbis | 2D/3D |

限频规则：

- 同一个枪声音效：0.05 到 0.12 秒内最多播一次。
- 小爆炸：0.08 秒内最多播一次。
- 大爆炸和 Boss 技能：允许打断普通限频。
- 同时播放 AudioSource 总数建议小于 48。

推荐音效来源：

- 爆炸、武器、冲击：Sonniss GDC。
- UI、whoosh、短提示：Pixabay。
- 怪物特殊叫声：Freesound，但只选 CC0 或可商用 CC-BY。

## 9. 场景和后处理搭建

第一版战场：

```text
SceneRoot
  Lighting
    Directional Light
    Reflection Probe
  Battlefield
    Ground
    Road
    HumanBase
    OrcBase
    Obstacles
    Decoration
  Units
  Projectiles
  Effects
  Audio
  UI
```

光照：

- Directional Light 角度：`X=45, Y=-35, Z=0`。
- Ambient Color：偏冷灰蓝。
- 兽族侧加少量红色点光或雾色。
- 人族侧用蓝色/白色强调科技感。

地面：

- `ambientCG` 找 `Dirt / Ground / Rock` 材质。
- 做一张低对比度地面材质，避免与单位抢视觉。
- 爆炸区域可用 Decal 或扁平透明 mesh 做焦痕。

镜头：

- 直播竖屏：固定 720x1280 或 1080x1920。
- 主镜头看全场，技能时短暂震屏。
- 震屏只用于大爆炸和 Boss 技能，幅度小于 0.25 秒。

## 10. 素材导入验收清单

每导入一个资源包，必须检查：

- 资源是否来自上表官方地址。
- 是否保存许可证记录。
- 模型是否能在 Unity 中正常显示。
- 模型尺寸是否统一到项目单位。
- 模型朝向是否与战场移动方向一致。
- 动画是否能播放。
- 材质是否丢失或变粉。
- 特效是否能自动停止并回池。
- 粒子碰撞是否只在必要子特效上开启。
- 音效是否进入 AudioMixer 对应分组。
- 打包后资源是否仍能加载。
