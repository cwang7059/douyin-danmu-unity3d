# Electron 手机控制面板接入记录

更新时间：2026-05-28

## 目标

- 做一个类似手机端直播平台的实体按钮面板。
- 面板通过本地 HTTP 弹幕网关控制 Unity 战场。
- 支持增加士兵、坦克、直升机、怪物，以及触发技能、治疗、加速和礼物。

## 实现

- 新增独立 Electron 工程：`tools/control-panel`。
- 主进程负责向 Unity 网关发起 HTTP 请求，避免浏览器渲染进程遇到跨域或 `file://` 限制。
- 渲染界面采用手机宽度布局，默认连接 `http://127.0.0.1:8765`。
- 数量按钮 `1 / 5 / 10` 采用多次发送，而不是只把数量塞进 `value`，避免 Unity 当前按单条命令处理时被忽略。
- 游戏侧新增 `aircraft` 结构化生成命令，按钮可直接生成或补充直升机单位。

## 坑点记录

- 不要让 Electron 渲染进程直接 `fetch` 本地网关。`file://` 页面和 CORS 配置容易导致请求失败，统一走主进程 IPC 更稳定。
- 不要把 Electron 项目放在 Unity 根目录直接安装依赖，`node_modules` 会非常大。当前放在 `tools/control-panel`，并在 `.gitignore` 中忽略 `node_modules/`。
- Unity 侧不能只依赖 `command.value` 增加数量。面板会按数量重复发送请求，和真实直播平台礼物连击更接近。
- 单位生成需要有上限。连续点击按钮时，士兵、坦克、直升机、怪物都从预热池里取对象，超过上限会回退到原有治疗/强化逻辑，避免同屏单位失控。
- 不要在战斗中为弹幕新增单位现场 `new GameObject`、`Instantiate` 模型或绑定动画。单位池已经在 loading 阶段预创建到上限，战斗中只允许激活 inactive 单位，否则每次刷礼物都会出现主线程卡顿。
- 生成单位时会同时播放召唤粒子。特效管理器也要预热常用效果，否则单位本身不卡了，第一次播放 `HumanSummon` / `OrcSummon` 仍然会因为创建粒子系统而顿一下。
